计算机网络知识点整理(2)：应用层

Last updated on March 22, 2025 pm

这是SJTU-CS3611《计算机网络》课程的知识点整理系列。本文整理部分为“第2章：应用层”。

一、应用层协议基本原理

1. 网络应用体系结构

  • ​C/S架构
    • 服务器：永久在线、固定IP、集中式管理（Web/FTP/Email）
    • 客户端：间歇性连接、动态IP、不直接通信
  • ​P2P架构
    • 无中心服务器、节点兼具客户端与服务端角色（BitTorrent/Skype）
    • 优势：自扩展性强、成本低
    • 挑战：动态IP、管理复杂

2. 进程通信

  • ​套接字（Socket）​
    • 应用层与传输层接口，通过IP+端口标识进程（HTTP:80/SMTP:25）
    • 关键概念：面向连接（TCP） vs 无连接（UDP）

3. 运输层服务选择

特性 TCP UDP
可靠性 可靠（重传机制） 不可靠
连接方式 面向连接（三次握手） 无连接
流量控制 有（滑动窗口） 无
拥塞控制 有（慢启动/拥塞避免） 无
适用场景 Web/Email/文件传输 实时音视频/DNS/SNMP

二、HTTP协议（重点！）

1. 核心机制

  • ​无状态性

    • 服务器不保存客户端状态，通过Cookie维持会话
    • ​Cookie工作流程：Set-Cookie头 → 客户端存储 → 后续请求携带Cookie

  • ​持久连接

    • HTTP/1.1默认，减少TCP握手开销
    • 对比非持久连接：每个对象需单独连接（2RTT → 1RTT优化）

2. 报文结构

  • ​请求报文

    • 方法：GET（获取资源）/POST（提交数据）/HEAD（获取头部）

  • ​响应报文

    • 状态码：200（成功）/301（重定向）/404（未找到）/503（服务不可用）

3. 缓存机制（高频考点！）

  • ​Web缓存作用

    • 降低延迟、减少带宽消耗
    • 条件GET：If-Modified-Since头验证资源更新

  • ​缓存命中率计算

    • 公式：总延迟 = 命中率×缓存延迟 + (1-命中率)×源服务器延迟

4. HTTP演进

  • ​HTTP/2
    • 多路复用（解决队头阻塞）、头部压缩、服务器推送
  • ​HTTP/3
    • 基于QUIC（UDP实现可靠传输）、0-RTT连接建立

三、电子邮件系统

1. 核心协议

  • ​SMTP（发件协议）​
    • 命令交互式（HELO/MAIL FROM/RCPT TO/DATA）
    • 7位ASCII限制，Base64编码附件
  • ​POP3/IMAP（收件协议）​
    • POP3：下载即删除，无状态
    • IMAP：服务器存储状态，支持文件夹管理

2. 邮件报文格式

  • MIME扩展：支持多媒体附件（Content-Type/Content-Transfer-Encoding）

四、DNS系统

1. 分层结构

  • ​根DNS（13组集群）​ → ​TLD（.com/.org）​ → ​权威DNS（域名注册）​ → ​本地DNS

2. 查询过程

  • ​递归查询：本地DNS全权代理（客户端→本地DNS→根→TLD→权威）
  • ​迭代查询：每一步返回下一跳地址

3. 资源记录类型

  • ​A记录​（域名→IPv4）
  • ​AAAA记录​（域名→IPv6）
  • ​CNAME​（别名→规范名）
  • ​MX记录​（邮件服务器）

4. DNS安全

  • ​DDoS攻击：针对根/TLD服务器
  • ​DNSSEC：数据签名防止欺骗

五、P2P与文件分发

1. BitTorrent机制

  • ​Tracker服务器：协调节点列表
  • ​Tit-for-Tat策略：优先服务高速上传节点
  • ​Chunk选择：最稀缺优先（rarest first）

2. 分发效率对比

  • ​C/S模型：时间 ∝ N（客户端数）
  • ​P2P模型：时间 ∝ max(NF/us, F/dmin)

六、视频流与CDN

1. DASH技术

  • ​自适应码率：动态选择分辨率（带宽探测 → Manifest文件选择）

2. CDN部署策略

  • ​Enter Deep：深入ISP网络（Akamai）
  • ​Bring Home：集中骨干网（Limelight）
