计算机网络知识点整理(5)：网络层之控制平面

Last updated on May 9, 2025 pm

这是SJTU-CS3611《计算机网络》课程的知识点整理系列。本文整理部分为“第5章：网络层之控制平面”。

1. 路由算法

1.1 链路状态路由算法

核心思想

  • 全局拓扑视图：每个节点广播链路状态信息，全网同步。
  • Dijkstra算法：计算从源节点到所有其他节点的最短路径。

算法步骤

示例

步骤 N’ D(v) D(w) D(x) D(y) D(z)
0 {u} 2,u 5,u 1,u ∞ ∞
1 {u,x} 2,u 4,x - 2,x ∞
2 {u,x,y} - 3,y - - 4,y

优缺点

  • 优点：收敛快，无环路。
  • 缺点：洪泛链路状态信息导致高带宽消耗。

1.2 距离向量路由算法

核心思想

  • 分布式迭代：每个节点维护到所有目的地的距离向量。
  • Bellman-Ford方程：D_x(y) = min{c(x,v) + D_v(y)}

算法流程

  1. 节点定期向邻居发送距离向量。
  2. 收到邻居的更新后，重新计算本地距离向量。
  3. 若距离向量变化，通知邻居。

示例

问题与解决

  • 无穷计数问题：通过毒性逆转（Poisoned Reverse）解决，阻止环路。
  • 坏消息传播慢：采用水平分割（Split Horizon）。

1.3 LS与DV算法对比

特性 链路状态（LS） 距离向量（DV）
信息传播 全局洪泛链路状态 邻居间交换距离向量
计算位置 集中式（Dijkstra） 分布式（Bellman-Ford）
收敛速度 快（O(n^2)） 慢（可能振荡）
适用场景 大型网络 小型网络

2. 自治系统内部路由：OSPF

特点

  • 链路状态协议：洪泛LSA（链路状态通告），构建全网拓扑。
  • 分层结构：分区域（Area），骨干区域（Backbone）连接其他区域。
  • 认证机制：支持MD5验证，防止恶意路由注入。

工作流程

  1. 邻居发现：通过Hello报文建立邻居关系。
  2. LSA泛洪：广播链路状态信息。
  3. SPF计算：运行Dijkstra算法生成最短路径树。

应用场景

  • ISP内部网络：如谷歌B4网络使用OSPF实现高效流量调度。

3. 自治系统间路由：BGP

核心功能

  • 路径向量协议：通告AS路径（AS-PATH）和下一跳（NEXT-HOP）。
  • 策略路由：基于商业策略选择路径（如绕过竞争对手AS）。

消息类型

  1. OPEN：建立BGP对等体连接。
  2. UPDATE：通告/撤销路由。
  3. KEEPALIVE：维持连接。
  4. NOTIFICATION：报告错误。

路由选择策略

  1. 本地偏好（Local Pref）：优先选择高偏好值路径。
  2. 最短AS路径：减少跨AS跳数。
  3. 热土豆路由：选择出口成本最低的路径。

4. SDN控制平面

架构

  1. 数据平面：交换机执行“匹配-动作”规则。
  2. 控制平面：集中式SDN控制器（如OpenDaylight、ONOS）。
  3. 北向API：应用程序接口（如RESTful）。
  4. 南向API：控制器与交换机通信协议（如OpenFlow）。

OpenFlow协议

  • 消息类型：
    • Packet-in：未匹配流表项时上报控制器。
    • Flow-mod：控制器下发流表规则。
  • 流表项：匹配字段（IP、端口等）+ 动作（转发、丢弃、修改字段）。

示例：链路故障处理

  1. 交换机通过Port Status消息通知控制器。
  2. 控制器更新拓扑，重新计算路径。
  3. 下发新流表到受影响交换机。

5. ICMP协议

功能与消息类型

类型 代码 描述
0 0 回显应答（Ping响应）
3 0 目标网络不可达
8 0 回显请求（Ping请求）
11 0 TTL过期（Traceroute）

应用

  • Ping：检测主机可达性。
  • Traceroute：路径跟踪，利用TTL递减和ICMP超时消息。

6. 网络管理与配置

SNMP

  • 组件：
    • 管理站（Manager）：发起查询/设置请求。
    • 代理（Agent）：响应请求，上报Trap消息。
  • MIB：管理信息库，存储设备状态变量（如接口流量计数）。

NETCONF/YANG

  • NETCONF：基于XML的配置协议，支持事务操作。
  • YANG：数据建模语言，定义配置和状态数据结构。
